# Reschedule Rules

Use checkpoints to keep the plan useful after delays.

## Delay Levels

Light delay:

- Behind by <= 10% of total work time.
- Compress buffer but keep core milestones.

Moderate delay:

- Behind by 10-25% of total work time.
- Drop optional materials and polish tasks.
- Move directly toward the minimum deliverable.

Severe delay:

- Behind by > 25% of total work time or soft deadline is missed.
- Switch to minimum deliverable mode.
- Protect final export, submit, run, or presentation readiness.

Hard-deadline risk:

- Stop broad exploration.
- Finish the smallest complete artifact that can be submitted, shown, run, or used.

## Reschedule Method

When rescheduling:

1. Restate remaining time.
2. Identify the protected deliverable.
3. Cut optional tasks first.
4. Reduce polish before reducing correctness.
5. Preserve final check/export time.
6. Give the next action in one sentence.

## Minimum Deliverable

Define a minimum deliverable for every plan.

Good minimum deliverables are:

- Complete enough to submit or continue from
- Smaller than the ideal version
- Specific enough to verify
- Focused on the user's actual deadline

Examples:

- For writing: complete structure and core claims, even if wording is rough.
- For slides: title, background, main evidence, conclusion, and export.
- For code: the main path works and has one focused verification.
- For reading: usable notes tied to the next task.
- For revision: highest-risk sections fixed and final file saved.
